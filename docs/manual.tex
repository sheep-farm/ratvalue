\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

\textbf{ratvalue} is a C++23 library implementing Aswath Damodaran's complete
valuation framework, from earnings normalization to specialized models for
startups, distressed firms, and financial institutions.
The project is built on top of \texttt{ratmoney}, which provides exact rational
monetary arithmetic (\texttt{int64\_t} + GCD over \texttt{\_\_int128}).

This manual covers:
\begin{itemize}
  \item Theoretical foundations of each valuation model;
  \item Complete C++23 API specification;
  \item Verifiable numerical examples;
  \item Real-world case study with Petrobras (PETR4) and Itaú Unibanco (ITUB4).
\end{itemize}

Prerequisites: familiarity with C++17+, basic linear algebra, and corporate
finance at the undergraduate level.

\section*{Step 3 --- Minimal DCF in C++}

\begin{lstlisting}[style=cpp, caption={Minimal two-stage DCF --- C++}]
#include "fcff.h"
#include "capm.h"
#include "wacc.h"
#include "dcf.h"
#include "detail.h"

using namespace ratvalue;
using ratmoney::Currency, ratmoney::Rational, ratmoney::CurrencyDescription;

int main() {
    CurrencyDescription BRL{"BRL", "R$", 2};
    Rational ONE{1, 1};
    auto B = [&](int64_t b) {          // b = billions
        return Currency{b * 100'000'000'000LL, ONE, BRL};
    };

    // Cost of capital
    Rational ke = *cost_of_equity({
        .risk_free_rate      = {1, 20},   // 5%
        .beta                = {13, 10},  // 1.30
        .equity_risk_premium = {1, 20},   // 5%
    });
    Rational wacc = *compute_wacc({
        .equity_weight  = {2, 3},
        .debt_weight    = {1, 3},
        .cost_of_equity = ke,
        .cost_of_debt   = {6, 100},  // 6%
        .tax_rate       = {34, 100},
    });

    // FCFF base and projection: 3% for 5 years
    Currency fcff0 = *compute_fcff({
        .ebit                      = B(126),
        .tax_rate                  = {34, 100},
        .depreciation_amortization = B(45),
        .capex                     = B(70),
        .delta_nwc                 = B(5),
    });
    auto fcffs = *project_fcff({
        .base_fcff = fcff0,
        .stages    = {{{3, 100}, 5}},
    });

    // DCF
    auto r = *compute_dcf({
        .projected_fcff     = fcffs,
        .wacc               = wacc,
        .terminal_growth    = {3, 200},   // 1.5%
        .net_debt           = B(250),
        .shares_outstanding = 13'000'000'000LL,
    });

    double price = detail::to_double(r.equity_value_per_share);
    // -> ~39.88
}
\end{lstlisting}

\chapter{Introduction}

\section{Motivation}

Valuation models are the analytical core of every capital allocation decision.
Damodaran's framework, developed over three decades in \textit{Investment
Valuation} (3rd ed., 2012) and \textit{The Dark Side of Valuation} (3rd ed.,
2018), is the most rigorous and comprehensive reference available.

Existing implementations focus on Python or spreadsheets, which have two
significant shortcomings for serious quantitative analysis:

\begin{enumerate}
  \item \textbf{Numerical precision:} \texttt{float64} cannot represent decimal
        fractions exactly. Errors accumulate across chained calculations.
  \item \textbf{Performance:} Python has the GIL and boxing overhead;
        spreadsheets are neither version-controlled nor auditable as code.
\end{enumerate}

\textbf{ratvalue} addresses both: exact monetary arithmetic via
\texttt{ratmoney}, and compiled C++ performance with \texttt{-O2}.

\chapter{Infrastructure: ratmoney and Internal Types}

\section{ratmoney: Exact Monetary Arithmetic}

Every monetary unit in \textbf{ratvalue} is represented by the type
\texttt{ratmoney::Currency}, which stores:

\begin{itemize}
  \item \texttt{units}: count in the smallest denomination (\emph{cents},
        for BRL and USD with \texttt{precision=2});
  \item \texttt{rate}: the value of 1 unit of this currency in units of an
        implicit base currency (type \texttt{Rational});
  \item \texttt{description}: name, symbol, and decimal precision.
\end{itemize}

The type \texttt{ratmoney::Rational} represents a rational number $p/q$ in
canonical form: $q > 0$, $\gcd(|p|, q) = 1$. All arithmetic uses
\texttt{\_\_int128} as an intermediate, eliminating silent overflow.

\begin{lstlisting}[style=cpp, caption={Creating currency values in ratvalue}]
ratmoney::CurrencyDescription BRL{"BRL", "R$", 2};
ratmoney::Rational UNIT_RATE{1, 1};

// BRL 1 billion = 1e11 cents
Currency B(int64_t billions) {
    return Currency{billions * 100'000'000'000LL, UNIT_RATE, BRL};
}

Currency ebit = B(145);   // BRL 145 billion
\end{lstlisting}

\begin{atencao}
Currency identity is determined by \texttt{CurrencyDescription}.
Two \texttt{Currency} objects can only be added or compared if they share the
\emph{same description}. The \texttt{rate} field is for \emph{currency
conversion}, not for denomination identification.
ratvalue validates currency consistency in all relevant functions and returns
\texttt{ValuationError::CurrencyMismatch} on violation.
\end{atencao}

\section{types.h: Errors and Projection}

\begin{lstlisting}[style=cpp, caption={Core types in ratvalue}]

\chapter{Earnings Normalization}

\section{Theory}

Cyclical firms (oil, mining, steel, agriculture) exhibit volatile EBIT that
distorts any DCF based on the most recent year. Damodaran (ch.~8) recommends
normalizing before estimating value, using one of two approaches:

\begin{description}
  \item[Historical average]
    Computes the arithmetic mean of EBIT over the last $N$ years ($N \geq 2$),
    capturing the ``mid-cycle EBIT'':
    \[
      EBIT_{\text{norm}} = \frac{1}{N} \sum_{i=1}^{N} EBIT_i
    \]

  \item[Normalized margin]
    Applies a historical (or sector-level) operating margin to current revenues.
    Useful when revenues are stable but margins fluctuate:
    \[
      EBIT_{\text{norm}} = \text{Current Revenue} \times \text{Normalized Margin}
    \]
\end{description}

\section{API}

\begin{lstlisting}[style=cpp]
enum class NormalizationMethod {
    HistoricalAverage,  // method 1
    NormalizedMargin,   // method 2
};

struct NormalizationInputs {
    NormalizationMethod             method;
    std::vector<ratmoney::Currency> historical_ebit;   // >= 2 for Average
    ratmoney::Currency              current_revenue;    // for Margin
    ratmoney::Rational              normalized_margin;  // for Margin
};

expected<Currency, ValuationError>
normalize_ebit(const NormalizationInputs& inputs);
\end{lstlisting}

\begin{exemplo}{Petrobras --- historical cycle average}
\begin{lstlisting}[style=cpp, numbers=none]
auto result = normalize_ebit(NormalizationInputs{
    .method = NormalizationMethod::HistoricalAverage,
    .historical_ebit = {B(80), B(95), B(145), B(165), B(145)},
    // EBIT 2019-2023 in BRL billions
    .current_revenue = B(500),  // not used in this method
    .normalized_margin = {0, 1},
});
// result->units() / 1e11 = BRL 126B (average)
\end{lstlisting}
\end{exemplo}

\section{Remarks}

The arithmetic uses accumulated \texttt{Currency::add} followed by
\texttt{Currency::scale} with factor $\{1, N\}$. Rounding is
\textit{HalfEven} (banker's rounding), so the exact sum is preserved and
the rounding error is at most 1 cent.

\chapter{Cost of Capital}

\section{CAPM with Country Risk Premium}
\label{sec:capm}

\subsection{Model}

For emerging markets, Damodaran (\textit{Investment Valuation}, ch.~7)
extends the CAPM by adding a country risk premium weighted by the firm's
exposure $\lambda$:

\begin{equation}
  \Ke = \Rf + \beta \cdot \ERP_{\text{mature}} + \lambda \cdot \CRP
  \label{eq:capm_crp}
\end{equation}

Where:
\begin{itemize}
  \item $\Rf$: global risk-free rate (US 10Y T-Bond, stripped of any
        differential inflation component if necessary);
  \item $\ERP_{\text{mature}}$: equity risk premium of the reference market
        (typically the US; Damodaran publishes annual estimates);
  \item $\CRP$: country risk premium, computed as the sovereign spread
        $\times$ ($\sigma_{\text{equity}} / \sigma_{\text{bond}}$);
  \item $\lambda$: firm's exposure to country risk.
        $\lambda = 1$ for firms with entirely domestic revenues;
        $\lambda < 1$ for exporters or multinationals.
\end{itemize}

\begin{definicao}{Country Risk Premium (CRP)}
  \[
    \CRP = \text{Sovereign spread} \times
           \frac{\sigma_{\text{equity, emerging market}}}
                {\sigma_{\text{sovereign bond}}}
  \]
  For Brazil in 2024: EMBI+ spread $\approx 2\%$;
  volatility ratio $\approx 1.5$; $\CRP \approx 3\%$.
\end{definicao}

\subsection{API}

\begin{lstlisting}[style=cpp]
struct CAPMInputs {
    Rational risk_free_rate;
    Rational beta;
    Rational equity_risk_premium;              // ERP of mature market
    Rational country_risk_premium{0, 1};       // CRP (default: 0)
    Rational lambda{1, 1};                     // country exposure (default: 1)
};

expected<Rational, ValuationError>
cost_of_equity(const CAPMInputs& inputs);
\end{lstlisting}

All computation uses exact rational arithmetic. For
$\Rf = 5\% = \frac{1}{20}$, $\beta = 1.30 = \frac{13}{10}$,
$\ERP = 7\% = \frac{7}{100}$, $\CRP = 3\% = \frac{3}{100}$, $\lambda = 1$:

\[
  \Ke
  = \tfrac{1}{20} + \tfrac{13}{10} \cdot \tfrac{7}{100} + 1 \cdot \tfrac{3}{100}
  = \tfrac{50}{1000} + \tfrac{91}{1000} + \tfrac{30}{1000}
  = \tfrac{171}{1000} = 17.1\%
\]

\section{Beta: Hamada and Bottom-Up}

\subsection{Hamada Equation}

Financial leverage amplifies systematic risk:

\begin{equation}
  \beta_L = \beta_U \cdot \left[1 + (1-t) \cdot \frac{D}{E}\right]
  \label{eq:hamada}
\end{equation}

Inversely:
\begin{equation}
  \beta_U = \frac{\beta_L}{1 + (1-t) \cdot D/E}
  \label{eq:hamada_inv}
\end{equation}

\subsection{Bottom-Up Beta}

Historical beta (return regression) is noisy and reflects the past capital
structure. Damodaran recommends the \emph{bottom-up beta} approach:

\begin{enumerate}
  \item Collect $\beta_L$ from $n$ comparable firms in the same sector;
  \item Unlever each comparable via eq.~\eqref{eq:hamada_inv};
  \item Compute the average: $\bar\beta_U = n^{-1}\sum_i \beta_{U,i}$;
  \item Re-lever to the target capital structure via eq.~\eqref{eq:hamada}.
\end{enumerate}

\subsection{API}

\begin{lstlisting}[style=cpp]
// Lever beta
expected<Rational, ValuationError>
lever_beta(Rational unlevered_beta, Rational debt_to_equity, Rational tax_rate);

// Unlever beta
expected<Rational, ValuationError>
unlever_beta(Rational levered_beta, Rational debt_to_equity, Rational tax_rate);

struct BottomUpBetaInputs {
    vector<Rational> comparable_levered_betas;
    vector<Rational> comparable_debt_to_equity;
    Rational         comparable_tax_rate;
    Rational         target_debt_to_equity;
    Rational         target_tax_rate;
};

expected<Rational, ValuationError>
bottom_up_beta(const BottomUpBetaInputs& inputs);
\end{lstlisting}

\begin{atencao}
The average of $\beta_U$ inside \texttt{bottom\_up\_beta} is computed in
\texttt{double}, as market betas are statistical estimates without exact
precision. The final re-levered result is returned as \texttt{Rational}
with 4 decimal places.
\end{atencao}

\section{Synthetic Cost of Debt}
\label{sec:kd_sint}

\subsection{Methodology}

Firms without traded bonds have no observable $\Kd$. Damodaran proposes
estimating it from the \emph{interest coverage ratio} (ICR):

\[
  \text{ICR} = \frac{\text{EBIT}}{\text{Interest Expense}}
  \quad\longrightarrow\quad
  \text{synthetic rating}
  \quad\longrightarrow\quad
  \Kd = \Rf + \text{spread}(\text{rating})
\]

The ICR $\to$ rating table is published annually at
\url{https://pages.stern.nyu.edu/~adamodar/}.
ratvalue implements the 2023 tables for \emph{large cap} and
\emph{small cap} (tighter thresholds for smaller firms).

\section{FCFE --- Free Cash Flow to Equity}

\subsection{Formula}

\end{equation}

Where $\delta = D/(D+E)$ is the debt ratio. The interpretation:
equity holders finance only the $(1-\delta)$ share of net reinvestment;
the rest is covered by new borrowing.

\section{Fundamental Growth}
\label{sec:growth}

\subsection{Theory}

Damodaran insists that growth assumptions must be \emph{anchored} in
fundamentals. ``Free'' growth (without reinvestment) is a common error
that artificially inflates value.

\begin{equation}
  g_{\text{firm}} = \ROIC \times \RR
  \label{eq:g_firma}
\end{equation}

\begin{equation}
  g_{\text{equity}} = \ROE \times (1 - \text{payout})
  \label{eq:g_equity}
\end{equation}

Where:
\[
  \ROIC = \frac{\NOPAT}{\IC} \qquad
  \RR   = \frac{\text{CapEx} - D\&A + \Delta NWC}{\NOPAT}
\]

\chapter{Discounted Cash Flow}

\section{Standard Model: FCFF and WACC}

\subsection{Two-Stage Structure}

Damodaran's standard DCF divides the horizon into two stages:

\begin{enumerate}
  \item \textbf{Explicit period} ($t = 1, \ldots, N$):
        projected FCFFs discounted individually.
  \item \textbf{Stable perpetuity} ($t > N$):
        terminal value (TV) computed via the Gordon Growth Model.
\end{enumerate}

\begin{align}
  \EV
  &= \sum_{t=1}^{N} \frac{\FCFF_t}{(1+\WACC)^t}
   + \frac{\TV_N}{(1+\WACC)^N}
  \label{eq:dcf_ev}
  \\[4pt]
  \TV_N
  &= \frac{\FCFF_N \cdot (1 + g)}{(\WACC - g)}
  \label{eq:gordon}
  \\[4pt]
  \text{Equity} &= \EV - \text{Net Debt}
  \\[4pt]
  P_{\text{share}} &= \frac{\text{Equity}}{n_{\text{shares}}}
\end{align}

\subsection{Implementation: double for Discounting, Rational for TV}

\begin{atencao}
The discount factor $(1+r)^{-t}$ is irrational for $t > 1$ in most cases.
ratvalue uses \texttt{double} \emph{only} for exponentiation; all other
operations are exact rationals. In particular, the TV multiplier is an exact
\texttt{Rational}:
\[
  \TV_{\text{mul}} = \frac{(1+g)/g_{\text{den}}}{(\WACC - g)}
  = \frac{(g_{\text{den}} + g_{\text{num}}) \cdot \WACC_{\text{den}}}
         {\WACC_{\text{num}} \cdot g_{\text{den}} - g_{\text{num}} \cdot \WACC_{\text{den}}}
\]
\end{atencao}

\subsection{API}

\section{H-Model (Fuller \& Hsia, 1984)}

The H-Model is a closed-form solution for growth that \emph{declines

\end{equation}

temporal discounting is involved.

\chapter{Terminal Value}

\section{Consistency with ROIC}

\subsection{The Problem with the Standard Gordon Model}

The Gordon Growth TV implies a reinvestment rate:
\[
  \RR_{\text{impl}} = \frac{g}{\ROIC_{\text{stable}}}
\]

If the analyst assumes $g = 3\%$ but $\ROIC_{\text{stable}} = 8\%$, then
$\RR_{\text{impl}} = 37.5\%$. If the projected FCFFs used $\RR = 30\%$,
there is a discontinuity at the terminal stage that inflates the TV.

\subsection{Consistent Terminal Value (Damodaran ch.~12)}

\begin{equation}
  \TV = \frac{\NOPAT_{\text{stable}} \cdot (1 - g/\ROIC_{\text{stable}})}{
              \WACC - g}
  \label{eq:tv_consistente}
\end{equation}

Uses the terminal-period NOPAT as the base, deriving the stable FCFF from the
reinvestment rate consistent with the assumed ROIC.

\subsection{Consistency Audit}

};

    Rational wacc, Rational terminal_growth, Rational terminal_roic);

\end{lstlisting}

\begin{atencao}
When $\ROIC_{\text{stable}} < \WACC$, the TV is still positive but the firm
destroys value in the perpetuity (\texttt{value\_creating = false}).
This is economically valid; the model does not reject this case.
\end{atencao}

\chapter{APV --- Adjusted Present Value}

\section{Motivation}

The \APV\ explicitly separates the unlevered firm value from the tax shield
benefit:

\begin{equation}
  \APV = V_U + PV(\text{Tax Shields})
\end{equation}

Where $V_U$ is the unlevered firm value, obtained by discounting FCFFs at the
unlevered cost of equity $\Ku$:

\[
  \Ku = \Rf + \beta_U \cdot \ERP + \lambda \cdot \CRP
\]

APV is preferred over WACC when the capital structure changes over time
(LBOs, recapitalizations, projects with amortization schedules).

\section{Modigliani-Miller (1963): Permanent Debt}

For permanent debt and risk-free tax shields (discounted at $\Rf$):

\begin{equation}
  PV(\text{TS})_{\text{MM}} = t \cdot D
  \label{eq:ts_mm}
\end{equation}

\begin{lstlisting}[style=cpp]
struct APVInputs {
    std::vector<Currency> projected_fcff;
    Rational              terminal_growth;
    Rational              unlevered_cost_of_equity;  // Ku
    Currency              debt_market_value;          // D
    Rational              tax_rate;                  // t
    Currency              net_debt;
    int64_t               shares_outstanding;
};

expected<APVResult, ValuationError>
compute_apv(const APVInputs& inputs);
\end{lstlisting}

\section{Miles-Ezzell (1980): Rebalanced Debt}

When the firm maintains a constant $D/V$ ratio (rebalanced each period),
tax shields bear the risk of assets and must be discounted at $\Ku$,
except the first period (known with certainty):

\begin{equation}
  PV(\text{TS})_{\text{ME}}
  = \frac{t \cdot \Kd \cdot D \cdot (1 + \Ku)}{(1 + \Kd)(\Ku - g)}
  \label{eq:ts_me}
\end{equation}

The \textbf{Miles-Ezzell multiplier} is computed in exact rational arithmetic
via \texttt{detail::r\_mul} and \texttt{detail::r\_div}.

\begin{lstlisting}[style=cpp]

    Rational cost_of_debt;   // required for ME formula
};

compute_apv_miles_ezzell(const APVMilesEzzellInputs& inputs);
\end{lstlisting}

\begin{exemplo}{MM vs.\ ME comparison (Petrobras)}
$D = \text{R\$}250\text{B}$, $t = 34\%$, $\Kd = 5.78\%$,
$\Ku \approx 11.57\%$, $g = 1.5\%$:
\[
  PV(\text{TS})_{\text{MM}} = 0.34 \times 250 = \text{R\$}85\text{B}
\]
\[
  PV(\text{TS})_{\text{ME}}
  = \frac{0.34 \times 0.0578 \times 250 \times 1.1157}{1.0578 \times 0.1007}
  \approx \text{R\$}51\text{B}
\]
MM overstates the tax shield benefit by $\approx 40\%$ in this case.
\end{exemplo}

\chapter{EVA and Excess Returns}

\section{Theoretical Background}

The excess returns model (Damodaran, ch.~13) decomposes firm value into:

\begin{equation}
  V = \IC + PV(\EVA)
  \label{eq:eva_decomp}
\end{equation}

Where $\EVA_t = (\ROIC_t - \WACC) \times \IC_t$ and $\IC_t$ grows with
reinvestment. For the single-stage (Gordon) stable case:

\begin{equation}
  V = \IC \cdot \frac{\ROIC - g}{\WACC - g}
  \label{eq:eva_single}
\end{equation}

\textbf{DCF equivalence:} when inputs are consistent
($g = \ROIC \times \RR$ and $\FCFF = \NOPAT(1-\RR)$),
equations~\eqref{eq:dcf_ev} and \eqref{eq:eva_decomp} yield the same $V$.
The EVA decomposition reveals \emph{value of assets in place} vs.\
\emph{value of future growth}.

\section{Economic Interpretation}

\begin{itemize}
  \item $\ROIC > \WACC$: the firm creates value above its cost of capital;
        $PV(\EVA) > 0$.
  \item $\ROIC = \WACC$: \emph{neutral} growth; $V = \IC$.
  \item $\ROIC < \WACC$: every dollar invested destroys value; $PV(\EVA) < 0$.
\end{itemize}

\section{API}

\chapter{Justified Multiples}

\section{Background}

Market multiples (P/E, P/BV, EV/EBITDA) are frequently interpreted without

Where $g_{\text{equity}} = \ROE \times (1 - \text{payout})$ and
$\RR = g/\ROIC$ (firm reinvestment rate).

\chapter{Optimal Capital Structure}

\section{Algorithm}

The minimum cost of capital maximizes firm value. ratvalue sweeps
debt ratios $d \in \{0/N, 1/N, \ldots, (N-1)/N\}$ and for each point:

\begin{enumerate}
  \item Computes $D/E = d/(1-d)$ as \texttt{Rational};
  \item Levers $\beta_U \to \beta_L$ via Hamada (eq.~\ref{eq:hamada});
  \item Computes $\Ke$ via CAPM (eq.~\ref{eq:capm_crp});
  \item Iterates $\Kd$ from the ICR:
        \begin{enumerate}
          \item $\text{Interest} = D \times \Kd$
          \item $\text{ICR} = \text{EBIT} / \text{Interest}$
          \item $\Kd \leftarrow \Rf + \text{spread}(\text{ICR})$
          \item Repeat until convergence ($< 10^{-9}$, typically 2--5 iterations)
        \end{enumerate}
  \item Computes exact $\WACC$ as \texttt{Rational}.
\end{enumerate}

The optimum is the point of minimum $\WACC$.

\chapter{Specialized Models}

\section{Scenario-Based Valuation: Startups and Transitioning Firms}

\subsection{Motivation}

Young firms or those undergoing technological transitions have strongly
asymmetric outcome distributions. A single ``base case'' DCF ignores this
dispersion. Damodaran (\textit{Dark Side}, ch.~10--12) recommends:

\begin{enumerate}
  \item Define $n$ scenarios (typically 3--5) with probabilities;
  \item Compute the TV of each scenario: $\TV_i = \FCFF_i \cdot \text{mul}_i$;
  \item Discount each TV to the present: $PV_i = \TV_i / (1+\WACC_i)^T$;
  \item Apply a survival probability:
\end{enumerate}

\begin{equation}
  \EV_{\text{adjusted}} = \left[\sum_i P_i \cdot PV_i \right]
                          \cdot P(\text{survive})
                        + V_{\text{failure}} \cdot [1 - P(\text{survive})]
\end{equation}

\subsection{API}

\section{Equity as an Option: Merton Model}

\subsection{Background}

In firms with high debt, equity behaves as a \emph{call option} on the firm's
assets, with a strike equal to the face value of debt (Merton, 1974):

\begin{align}
  d_1 &= \frac{\ln(V/D) + (r + \sigma_V^2/2)T}{\sigma_V \sqrt{T}} \\[4pt]
  d_2 &= d_1 - \sigma_V \sqrt{T} \\[4pt]
  E   &= V \cdot \mathcal{N}(d_1) - D \cdot e^{-rT} \cdot \mathcal{N}(d_2)
  \label{eq:merton}
\end{align}

The \emph{risk-neutral} probability of default is $\mathcal{N}(-d_2)$.
The distance to default $d_2$ is the primary indicator of financial health.

\begin{atencao}
$V$ and $\sigma_V$ are firm asset parameters, \emph{not directly observable}.
The iterative estimation (solving $\{E, \sigma_E\}$ from the nonlinear system
$E = \text{BS}(V, \sigma_V)$ and $\sigma_E E = \mathcal{N}(d_1) \sigma_V V$)
is not implemented; the user provides $V$ and $\sigma_V$ directly.
\end{atencao}

\subsection{API}

\begin{lstlisting}[style=cpp]

    double asset_volatility;  // sigma_V

};

struct EquityAsOptionResult {
    double equity_value;

    double d1;

\end{lstlisting}

The normal CDF is implemented via:
$\mathcal{N}(x) = \tfrac{1}{2}\,\mathrm{erfc}\!\left(-x/\sqrt{2}\right)$
(\texttt{std::erfc} from \texttt{<cmath>}).

\section{FCFE for Financial Institutions}

\subsection{Why Banks Are Different}

\begin{itemize}
  \item Debt (deposits, certificates) is \emph{raw material}, not a source of
        capex financing --- computing FCFF or WACC makes no economic sense.
  \item Reinvestment is regulated: Basel III requires minimum capital against
        risk-weighted assets (RWA).
  \item The only relevant cost of capital is $\Ke$.
\end{itemize}

\subsection{Model}

\begin{equation}
  \FCFE_{\text{bank}} = NI \times (1 - \RR_{\text{equity}})
\end{equation}

\[
  \RR_{\text{equity}} = \frac{\Delta \text{RWA} \times \text{target Tier1}}{NI}
\]

\subsection{API}

\begin{lstlisting}[style=cpp]
struct BankFCFEInputs {
    Currency net_income;
    Rational equity_reinvestment_rate;
};

expected<Currency, ValuationError>
compute_bank_fcfe(const BankFCFEInputs& inputs);

struct RegulatoryCapitalInputs {
    Currency net_income;
    Currency current_rwa;
    Currency projected_rwa;
    Rational target_tier1_ratio;
};

expected<Rational, ValuationError>
bank_equity_reinvestment_rate(const RegulatoryCapitalInputs& inputs);
\end{lstlisting}

For a bank's DCF, use \texttt{compute\_fcfe\_dcf} with FCFEs generated by
\texttt{compute\_bank\_fcfe}.

\section{Cross-Model Consistency}

For FCFF-DCF and FCFE-DCF to yield the same share price:
\begin{enumerate}
  \item $\FCFE = \FCFF - \text{Interest}(1-t) + \Delta D$;
  \item Both must use the same growth rate assumption;
  \item $\Delta D$ must be consistent with the $D/V$ ratio over time.
\end{enumerate}

For APV and WACC-DCF to yield the same $\EV$:
\begin{itemize}
  \item WACC assumes constant leverage (adjusts WACC each period);
  \item APV assumes a predetermined debt schedule.
  \item In general, APV is more flexible; WACC is simpler for stable structures.
\end{itemize}

\chapter{Python Bindings}

\section{Overview}

The \texttt{python/} directory provides nanobind bindings that expose the
complete ratvalue API to Python without rewriting any logic. All 18 modules
are covered. The C++ library performs every calculation; Python is only the
calling layer.

\begin{atencao}
The Python boundary converts \texttt{ratmoney::Currency} and
\texttt{ratmoney::Rational} to and from \texttt{double}. This introduces
floating-point rounding at the interface, but all internal arithmetic
remains exact. For production-grade auditing use the C++ API directly.
\end{atencao}

\section{Installation}

\textbf{Prerequisites:}
\begin{itemize}
  \item Python $\geq$ 3.9
  \item nanobind $\geq$ 2.0 and scikit-build-core $\geq$ 0.9
  \item The C++ library and ratmoney already installed in \texttt{/usr/local}
\end{itemize}

\textbf{Option 1 --- pip (recommended):}
\begin{lstlisting}[style=cpp, language=bash, numbers=none]
pip install nanobind scikit-build-core
pip install --no-build-isolation .
\end{lstlisting}

The \texttt{--no-build-isolation} flag is required so that pip uses the
already-installed \texttt{nanobind} and \texttt{scikit-build-core} instead
of attempting to fetch them into an isolated subprocess environment.
This is necessary on systems with PEP~668 restrictions (e.g.\ Arch Linux);
on standard environments it is optional.

\textbf{Option 2 --- manual CMake build:}
\begin{lstlisting}[style=cpp, language=bash, numbers=none]
cmake -S . -B build_py \
  -DCMAKE_BUILD_TYPE=Release \
  -DBUILD_PYTHON=ON \
  -DCMAKE_PREFIX_PATH="/usr/local"
cmake --build build_py -j$(nproc)
# shared library in build_py/python/ratvalue.cpython-*.so
\end{lstlisting}

The nanobind cmake directory is located automatically via the active Python
interpreter inside \texttt{python/CMakeLists.txt}; no manual path is needed.

inline Currency b2c(double billions) {
